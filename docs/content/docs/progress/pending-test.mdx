# 待测试

- KIE Seedance 2：配置页 KIE 内置视频模型应新增 `bytedance/seedance-2`、`bytedance/seedance-2-fast`、`bytedance/seedance-2-mini`；画布生成配置选择这些模型时应显示分辨率、比例、时长、生成声音，不显示 Kling motion-control 参数和水印；生成请求体应映射 `reference_image_urls` / `reference_video_urls` / `reference_audio_urls`，空参考素材不传空数组；未填写提示词时应按官方文档在提交前本地报错，参考音频单独使用时也应本地报错。
- KIE Seedance 2：请求体在没有真实 webhook 地址时不应发送空 `callBackUrl`；`bytedance/seedance-2` 可选择并发送 `4k`，`bytedance/seedance-2-fast` 和 `bytedance/seedance-2-mini` 不应显示或发送 `1080p` / `4k`。
- KIE Seedance 2：画布视频设置面板和摘要应按当前模型所属 KIE 渠道解析能力，即使全局默认调用格式不是 KIE，`bytedance/seedance-2` 也应显示 `4K`，旧节点残留智能时长时摘要应显示 `5s`。
- KIE Seedance 2：视频设置不应显示“智能”时长；如果旧节点残留 `videoSeconds=-1`，请求体应按官方默认回落为 `duration: 5`，不应再出现 `Invalid duration`。
- KIE 异步任务：当 `createTask` 返回业务错误且没有 `data.taskId` 时，失败卡片应显示接口返回的真实 `msg` / `message` / `error`，不应只显示“任务接口没有返回任务 ID”。
- KIE 异步视频任务：参考图 / 参考视频任务生成较久时前端应最多轮询 30 分钟；超时失败卡片应带任务 ID，便于去 KIE 后台对账；如果 `resultJson` 返回 `resultUrls`、`video_url`、`url` 等常见字段应能提取视频结果。
- 画布文本节点：文本内容超过节点高度时，鼠标悬停在文本区域应可直接用滚轮滚动，不应触发画布缩放。
- Canvas Agent：网页 SSE 连接失效后不应残留为可用客户端导致工具调用直接 500；重新连接画布后 `canvas_get_state` 应能正常返回当前画布状态。
- 画布生成配置：新建或未主动设置生成数量的生图配置节点应默认生成 1 张，不应继承全局历史生图数量导致一次出现多个结果节点。
- 画布参数设置弹窗：生成配置节点和节点快捷生成面板中的图像、视频、音频设置弹窗都应显示在节点右侧，并与触发节点保持 20px 间距。
- 模型渠道 Provider Adapter：配置页调用格式新增 Alibbit / KIE；“拉取模型”和“拉取全部”对 OpenAI / Gemini 继续远端拉取，对 Alibbit / KIE 使用内置模型目录；图像设置改为画质、清晰度、比例和生成数量四项，OpenAI / Gemini / Alibbit / KIE 应分别映射到各自请求参数；画布“生成配置”节点选择新渠道模型后应保持原有开始生成、停止、批量生图和错误提示交互。
- 模型可选项：生图 / 视频 / 文本 / 音频模型下拉应只显示对应能力的模型；视频模型下拉不应混入 `seedream-4-0` 等图片模型，当前已选的合法视频模型不应被回退成默认 `grok-imagine-video`。
- 画布生成配置视频模式：选择普通视频模型时仍显示原有尺寸 / 清晰度 / 秒数等参数；选择 KIE `kling-2.6/motion-control` 时只显示 `mode`、`character_orientation` 且两项必选，选择 `kling-3.0/motion-control` 时额外显示可选 `background_source`；这些动作参数应提供官方文档枚举选项，缺少人物图片、动作参考视频或 2.6 必填动作参数时应本地报错，提示词允许为空且空时不传 `prompt`。
- 画布视频重试：KIE Kling motion-control 生成失败后点击视频节点“重试”，即使原任务提示词为空也应继续按参考图、动作参考视频和动作参数重试，不应提示“找不到提示词”。
- 画布背景：暗色主题下画布背景应为 `#141414`，普通节点填充背景应为 `#252525`，浅色主题不受影响。
- 画布参数设置弹窗：图像、视频和音频设置应固定为 `340x460` 卡片，并使用一致的紧凑按钮、字体、间距和低对比未选中 / 高亮选中状态。
- 画布生成配置：点击“组装提示词”后面板应显示在生成配置节点右侧；长提示词应在固定高度编辑区内滚动，不再把卡片向下撑长。
- Alibbit 生图：选择 Alibbit 渠道下的 GPT Image 2 / Banana 2 模型族时，请求体中的 `model` 应按当前比例和清晰度映射为 `ali-gptimage2-*` / `ali-banana2-*` 实际模型，而不是直接发送 `gpt-image-2`。
- 画布生成配置：图像清晰度 1K / 2K / 4K 在生成配置节点和图片节点快捷生成面板中应可正常切换、保存和用于重试生成。
- 画布生成配置：1K 清晰度搭配 16:9、9:16、1:2、2:1 等非 1:1 比例生成图片时不应再因派生像素尺寸过低而直接失败。
- 本地 Agent 连接说明：新增提醒说明只有安装 Codex 插件或手动添加 MCP 后才会增加 Codex token 消耗，直接运行 `npx -y @basketikun/canvas-agent` 不会安装 MCP；需验证文案位置清晰。
- 本地 Agent 连接说明：画布面板和配置页 Codex Tab 改为区分“Codex 插件启动画布”和“直接运行 Agent 后网页连接”两种方式，需验证文案和布局清晰。
- 全站 Agent：Agent 面板从画布页抽离为全站常驻面板，挂在全局布局右侧，开/关只挤压左侧内容不遮挡；顶栏和画布工具条各有一个开关按钮，需验证在首页、生图、视频、素材、提示词库、画布等页面都能打开面板并保持 SSE 连接不断开。
- 全站 Agent：新增 `site_navigate` 工具，Agent 可直接跳转 `/`、`/canvas`、`/canvas/:id`、`/image`、`/video`、`/prompts`、`/assets`、`/config`；导航直接执行不走工具确认，需验证跳转正确且面板不消失。
- 全站 Agent：画布工具（canvas_apply_ops 等）仅在当前处于画布页时可用，不在画布页调用会返回“当前不在画布页，请先用 site_navigate 打开画布”，需验证 Agent 会先导航到画布再执行画布操作。
- 全站 Agent：Codex 会话改为单一站点级连续线程（不再按 canvasId 分），跨页面/跨画布保持同一条对话上下文，历史记录即这条线程；本地 Agent workspace 从按画布分改为固定 `site` workspace（`~/.infinite-canvas/codex-workspaces/site`），已装 `@basketikun/canvas-agent` 的用户需更新到新版本才生效。
- 全站 Agent：移除仅前端调用 OpenAI responses 接口的在线画布助手死代码（含 `requestToolResponse`），Agent 统一走 MCP + 本地 Codex 链路。
- 画布节点名称：每个节点保留 `title` 名称字段，默认名称显示在节点上方，双击名称可直接编辑，节点信息弹窗和 JSON 可查看该字段。
- 画布节点工具条：节点顶部工具条改为点击选中节点后显示，单纯鼠标移入 / 移出节点不应再频繁弹出。
- 画布组节点：组内子节点优先响应点击和拖拽；当鼠标位于组内节点上时，应移动 / 编辑该节点，而不是移动整个组。
- 画布组节点：工具栏新增“组”节点，普通节点拖入组区域会高亮目标组，松手后自动吸附进组；拖出组区域会解除归属，拖动组节点会一起移动组内子节点。
- GitHub Pages：新增版本 tag 触发的前端静态站点自动发布 workflow，需在仓库 Pages 设置中选择 GitHub Actions 并验证路由刷新回退。
- 画布图片切图：行数 / 列数会生成可直接拖拽的等分切图线，并支持新增横向 / 纵向线、删除单条线、重置切图线和切片数量预览。
- Codex App 插件文档：新增插件安装教程，区分 AI 自动安装、手动安装和本仓库开发调试安装；同步优化插件 README 与技能说明。
- 本地 Codex 连接画布文档：新增 Canvas Agent、MCP、SSE 与浏览器画布交互原理说明，需确认文档导航和描述是否清晰。
- 顶部导航：缩小桌面顶部导航栏高度，导航项下划线和内容垂直对齐需确认。
- 配置与用户偏好：新增独立顶部导航页面，并在页面和弹窗中复用同一套配置面板；新增 Codex Tab，展示插件安装和本地 Agent 启动步骤，可配置本地 Agent 地址、Connect token 和工具执行确认开关。
- Codex 全局连接：配置页可直接连接本地 Canvas Agent，连接状态同步到画布和顶部 OpenAI 状态图标，刷新后会按本地保存的地址和 token 自动尝试重连；需验证已连接、连接中、连接失败、断开和刷新自动重连状态。
- 画布 Agent：Codex 旧 thread 在重启后不可用时会自动新建会话并重试，需验证进入画布直接发送消息不会再出现 `thread not loaded` 或 `no rollout found`。
- 画布 Agent：切换不同画布时会清空并重新加载当前 canvasId 对应的 Codex 会话，需验证不会继续显示上一个画布的对话。
- 画布 Agent：右侧 Agent 面板移除网站 / 本机模式切换，固定使用本机 Agent。
- 配置与用户偏好：每个模型渠道新增 OpenAI / Gemini 调用格式选择，默认 OpenAI，切换默认格式时同步更新默认 Base URL；渠道 Tab 增加紧凑醒目的提醒，说明模型是否可选需要到“模型”Tab 选择。
