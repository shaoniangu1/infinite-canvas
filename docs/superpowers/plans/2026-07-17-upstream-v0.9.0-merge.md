# Upstream v0.9.0 Merge Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 将上游 `basketikun/infinite-canvas` 的 `v0.9.0` 完整合并到当前二开分支，同时保留并适配本地 Alibbit、KIE、Seedance、Kling motion-control 和生成配置节点能力。

**Architecture:** 以上游 `v0.9.0` 的画布插件、模型脚本、渠道模型对象和拆分后的画布组件为基线。本地 Alibbit/KIE 继续作为内置 Provider Adapter，由独立模型目录、模型能力配置和请求适配器承载；上游自定义模型脚本作为补充扩展机制，两者共用 `ChannelModel`、生成参数和媒体引用契约，不把 Provider 分支堆回页面组件。

**Tech Stack:** Vite、React 19、TypeScript、Ant Design、Tailwind、Zustand、Axios、localforage、Node.js test runner、Git。

## 3. 必须保留的本地能力

### 3.1 渠道和模型

- `ApiCallFormat` 继续支持 `openai`、`gemini`、`alibbit`、`kie`。
- 配置页“拉取模型”和“拉取全部”继续存在：OpenAI/Gemini 远端拉取，Alibbit/KIE 使用内置模型目录。
- Alibbit 内置 Banana 2、GPT Image 2 模型族，并按比例和清晰度解析为真实 `ali-*` 模型 ID。
- KIE 至少保留：
  - `kling-2.6/motion-control`
  - `kling-3.0/motion-control`
  - `bytedance/seedance-2`
  - `bytedance/seedance-2-fast`
  - `bytedance/seedance-2-mini`

### 3.2 生成请求

- Alibbit 生图使用 `POST /v1/videos` 提交、`GET /v1/videos/{id}` 轮询并读取 `url`；只有画布显式 `@` 引用的图片进入 `reference_images`。
- KIE 使用 `POST /api/v1/jobs/createTask` 提交、`GET /api/v1/jobs/recordInfo?taskId=...` 轮询；没有真实 webhook 时不发送空 `callBackUrl`。
- KIE 任务提交错误优先展示接口 `msg`、`message` 或 `error`；轮询最长 30 分钟，超时错误带 task ID；结果解析兼容常见 `resultJson` URL 形状。
- Kling 2.6 motion-control 必须上传人物图片和动作参考视频，支持空提示词；请求字段包含 `input_urls`、`video_urls`、必选 `character_orientation` 和必选 `mode`。
- Kling 3.0 motion-control 在 2.6 字段基础上增加可选 `background_source`。
- KIE Seedance 2 提示词必填，映射 `reference_image_urls`、`reference_video_urls`、`reference_audio_urls`，空引用数组不发送；仅音频输入在提交前拦截。
- KIE Seedance 2 不支持智能时长，旧值 `-1` 统一回落为 5 秒；完整模型支持 `480p/720p/1080p/4k`，Fast/Mini 仅支持 `480p/720p`。

### 3.3 参数和画布交互

- 图像参数保持“画质、清晰度、比例、生成数量”四项，由 Provider Adapter 映射为各服务实际字段。
- 视频参数由模型能力配置驱动，只显示当前模型相关字段，并在 UI 和提交层同时校验必填、枚举和引用素材。
- 生成配置节点的图像、视频、文本、音频设置浮层保持 `340 x 460`，定位节点右侧并间隔 20px；内容溢出可滚动且隐藏滚动条。
- 组装提示词面板显示在节点右侧，编辑区固定高度并可滚动。
- 生图配置未主动设置数量时默认 1 张，不继承全局历史数量。
- 视频失败节点重试保留空提示词 motion-control 场景，不错误提示“找不到提示词”。
- 文本节点长内容支持鼠标滚轮滚动；Canvas Agent 优先约束继续保留在 `AGENTS.md`。

## 4. 架构约束

- 上游文件拆分和插件架构优先，不恢复旧版巨型 `project.tsx`。
- Provider 目录只负责协议差异；模型目录只负责内置模型；模型能力配置只负责字段、选项和校验；页面组件只渲染能力配置。
- 自定义模型脚本与内置 Provider Adapter 共存：有脚本时执行脚本，无脚本时按内置调用格式分发。
- 自定义视频脚本必须能接收图片、视频和音频引用，不能只传参考图。
- 不增加旧本地存储字段兼容或迁移逻辑；项目仍在开发阶段，按合并后的新配置结构直接持久化。
- 不复制相同的 Provider 请求逻辑到 `image.ts`、`video.ts` 和组件；共享异步任务生命周期继续由独立运行时承载。
- 冲突处理优先保留上游实现，再以小范围导入和分发点接回本地能力，降低后续同步冲突。
